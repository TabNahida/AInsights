AInsights Index Methodology

AIndex is an alias for AInsights Index. Both names refer to the same default ranking method used on this site.

Frontier Groups and Weights

AIndex now uses a Frontier composite that gives the AA suite a 90% anchor, then allocates the remaining 10% across source-backed external benchmark evidence. The active group weights below sum to 100%.

AIndex Calculation

  1. For each benchmark, convert the raw score into a Best score ratio: model score divided by the highest score observed for that benchmark.
  2. Within each capability group, aggregate available benchmark ratios with Geometric Weighted Mean: exp(weighted average of log(ratio + 1)) - 1.
  3. Apply a light within-group evidence coverage discount, using scored fields divided by active fields raised to the power of 0.1; groups with only one scored field use the stricter 0.25 power.
  4. Aggregate available group scores with the group weights listed above.
  5. Apply the default missing policy: multiply by available group-weight coverage raised to the power of 0.25.
  6. Scale the resulting aggregate ratio by the highest AA Intelligence Index score available on the site, producing the displayed AIndex score.

Detailed Explanation

AIndex is designed to compare models across different benchmark scales without letting any single raw scoring convention dominate the final leaderboard. A percentage-style benchmark, an Elo-normalized field, and a long-context score can each have different ranges, so AIndex first places every active field on its own best-observed baseline.

The Frontier grouping keeps the AA suite as the primary stable anchor while allowing official release scores and public benchmarks to make smaller adjustments to frontier coding, tool-use, and reasoning comparisons. AA suite now carries 90% of AIndex, and the non-AA groups keep their previous relative balance inside the remaining 10%.

Official external benchmark rows are shared across variants of the same model, so a release table attached to a non-reasoning or reasoning tier can still inform the sibling tier instead of creating artificial gaps between variants.

The coverage discount is deliberately asymmetric: if a group has partial benchmark evidence, its group score is multiplied by a light within-group evidence coverage^0.1 discount; if that group has only one scored field, the stricter coverage^0.25 discount is used; if a model is missing whole groups, the final score is multiplied by available group-weight coverage^0.25. This keeps single-metric outliers in check without letting uneven external source coverage dominate the ranking.

The Custom weights panel exposes the same building blocks for comparison: Best score ratio or Raw score, Geometric Weight Mean or regular Weight Mean, available-only scoring, 0.25 coverage discount, sqrt coverage discount, weak prior, missing-as-zero, full-coverage gates, and per-benchmark weights.
