AInsights Index Methodology

AIndex is an alias for AInsights Index. Both names refer to the same default ranking method used on this site.

Regular Tests and Bonus Tests

AIndex now uses high-coverage regular tests for the base score, with extra emphasis on coding and hard reasoning, then adds a small source-backed bonus for hard external coding, math, and tool benchmarks. Regular weights sum to 100; bonus rows are not part of the missing-data denominator.

Terminal-Bench Hard, SciCode, Humanity's Last Exam, and CritPt use this calibrated curve before aggregation: log1p(5x) / log(6), where x is the best-score ratio.

LiveCodeBench is a high-weight regular coding signal. External benchmark:livecodebench scores can fill missing regular LiveCodeBench values through a linear fit learned from models that have both fields; existing regular LiveCodeBench values are never overwritten.

GDPval-AA, τ³-Banking, duplicate external HLE/GPQA rows, AIME 2026, HMMT, and external LiveCodeBench rows after fallback are not default bonus rows, to avoid rewarding models mainly for extra source coverage rather than stronger current capability.

AIndex Calculation

  1. For each regular benchmark, convert the raw score into a Best score ratio: model score divided by the highest score observed for that benchmark.
  2. Apply the regular-test calibration curve where configured, then aggregate available regular benchmark ratios with Geometric Weighted Mean: exp(weighted average of log(ratio + 1)) - 1.
  3. Apply the regular-test missing policy: multiply the base score by available regular-weight coverage raised to the power of 0.25.
  4. Scale the resulting base ratio by the highest AA Intelligence Index score available on the site.
  5. For each bonus benchmark with a score, add its best-score ratio times its share of the +2 bonus cap.
  6. Leave missing bonus rows out of the denominator, so absent hard-benchmark sources do not lower a model's base score.

Detailed Explanation

AIndex is designed to compare models across different benchmark scales without letting any single raw scoring convention dominate the final leaderboard. A percentage-style benchmark, an Elo-normalized field, and a long-context score can each have different ranges, so AIndex first places every active field on its own best-observed baseline.

The regular-test pool is now the primary anchor. It deliberately gives more weight to difficult coding and hard-problem benchmarks, while calibrating the hard-test high range so a cluster of strong scores does not over-expand small differences.

Official external benchmark rows are shared across variants of the same model, so a release table attached to a non-reasoning or reasoning tier can still inform the sibling tier instead of creating artificial gaps between variants.

For LiveCodeBench only, the site can synthesize a missing regular value from external benchmark:livecodebench data. The fit is learned from overlapping models in the current dataset, then clamped to the normal percentage range before scoring.

The coverage discount is deliberately asymmetric: regular tests use available regular-weight coverage^0.25, while bonus tests are excluded from the coverage denominator and only add points when source-backed scores exist. This keeps single-metric outliers in check without letting uneven external source coverage dominate the ranking.

The Custom weights panel exposes the same building blocks for comparison: Best score ratio or Raw score, Geometric Weight Mean or regular Weight Mean, available-only scoring, 0.25 coverage discount, sqrt coverage discount, weak prior, missing-as-zero, full-coverage gates, and per-benchmark weights.
